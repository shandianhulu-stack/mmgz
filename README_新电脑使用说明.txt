MIMO 工资评分生成器 Final V1.1

一、这个版本解决什么
1. 上传“米墨评分 Excel”后生成“网约工工资明细 Excel”。
2. 页面可以预览导出的 Sheet1。
3. 页面可以预览“最终金额”每个人每场评分明细，默认全部展开。
4. 导出 Excel 包含：Sheet1、最终金额、规则说明。
5. 默认使用“人工少量调分：严格不超发”。
6. 保留多种评分逻辑，可以在页面下拉切换。

二、保留的评分策略
1. 人工少量调分：严格不超发（默认，财务安全）
   多个100%，最后一行太低才拆成2-5行，任何人不微超。
2. 批次差值贴0：允许微超几分钱
   用于整批总差值尽量接近0，但可能有人多发0.01-0.04元，需财务确认后用。
3. 逐人四舍五入：单人差值最小
   每个人取最接近金额，可能微超。
4. 多行均衡高分
   多场一起分摊，评分普遍较高，但不像人工。
5. 最后一场补差
   旧逻辑，多个100%，最后一场补差，容易出现低分。

三、Windows 新电脑使用
1. 解压整个文件夹，不要只拖出某一个文件。
2. 先双击：环境检测_Windows.bat
3. 再双击：启动软件_Windows.bat
4. 浏览器自动打开本地页面。
5. 上传 Excel，点击“预览导出表”。
6. 确认后点击“生成并下载”。

四、Mac 新电脑使用
1. 解压整个文件夹。
2. 右键打开：环境检测_Mac.command
3. 右键打开：启动软件_Mac.command
4. 浏览器自动打开本地页面。
5. 如果提示没有权限，在终端进入文件夹执行：chmod +x *.command

五、依赖说明
本包自带 Excel 处理依赖 vendor/openpyxl，不需要 pip install，不需要虚拟环境。
电脑本身仍需要 Python 3.9 或更高版本。

六、输入文件
必选：米墨评分 Excel，字段至少包含：姓名、电话、金额。
建议必选：结算过程 / 工资明细表，用于回填预算、已完成场次、本次结算场次、最大可结算金额、是否足额。

七、注意
默认模式严格不超发，所以总体差值会是负数累计，这是正常结果。
如果为了总体差值更接近0切换到允许微超模式，必须先经过财务确认。


V1.4 安装版说明：
Windows 运营电脑优先使用 GitHub Actions 产物 MIMO_WageScore_Setup_V1.4.0.exe。
双击后按“下一步”安装，安装完成桌面会有 MIMO工资评分生成器 快捷方式。
